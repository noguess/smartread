## 1. V2.0 版本概述

### 1.1 版本背景
V1.3 版本已经建立了完整的学习闭环和核心算法体系。V2.0 版本将在保持现有功能架构不变的基础上，重点优化用户体验，使产品更贴合初中生（12-15岁）用户群体的使用习惯和审美偏好。

### 1.2 核心升级方向
本版本聚焦三大优化方向：

1. **🎨 青少年友好化设计**
   - 更活泼、更具亲和力的视觉风格
   - 符合初中生审美和心理特征的UI/UX
   - 降低学习压力感，提升使用趣味性

2. **🌍 国际化支持 (i18n)**
   - 支持中文/英文界面一键切换
   - 为海外用户和双语学习场景提供支持
   - 提升产品的适用范围和教育价值

3. **📖 阅读体验优化**
   - 更科学的排版和视觉层次
   - 更舒适的阅读节奏和交互流程
   - 护眼模式和个性化阅读设置

## 2. 详细需求规格

### 🎨 需求一：青少年友好化设计改造

#### 2.1 设计原则
针对初中生用户群体（12-15岁），设计需要遵循以下原则：

**心理特征适配：**
- **减少压迫感**：避免过于严肃的商务风格，采用轻松活泼的视觉语言
- **增强成就感**：通过视觉反馈强化学习正向激励
- **降低门槛**：简化操作流程，减少认知负担
- **趣味性平衡**：在专业性与趣味性之间找到平衡点

**审美偏好适配：**
- 色彩明快但不刺眼
- 图标生动但不幼稚
- 动效流畅但不浮夸
- 字体清晰但不呆板

#### 2.2 具体设计要求

##### 2.2.1 色彩系统优化
**主色调 (Primary Color)：**
- **从**：Deep Blue / Teal（沉稳专业）
- **到**：渐变蓝紫色系 (#4A90E2 → #7B68EE) 或活力橙蓝组合
- **理由**：渐变色更具现代感和活力，符合年轻用户审美

**辅助色 (Secondary Color)：**
- 增加明亮的强调色：如薄荷绿 (#00D9A5)、阳光黄 (#FFD93D)
- 用于按钮、徽章、进度条等正向反馈元素

**状态色优化：**
- ✅ **成功/掌握**：从单调的绿色改为带渐变的翠绿 (#10B981 → #34D399)
- ⏳ **学习中**：使用温暖的橙色渐变 (#F59E0B → #FBBF24)
- 🔴 **需复习**：保留红色但降低饱和度，避免过强的负面情绪

**背景系统：**
- 提供**浅色模式（默认）**和**护眼模式**切换
- 浅色模式：米白/浅灰柔和底色 (#F8F9FA)
- 护眼模式：暖色调米黄底 (#FFF8E7) + 降低对比度

##### 2.2.2 图标与插图系统
**图标风格：**
- 从 Material Icons Outlined 升级为 **Material Symbols Rounded**
- 增加圆角和笔触粗细，更友好亲和

**插图元素：**
- **空状态插图**：为以下场景添加轻量级插图
  - 暂无学习记录
  - 暂无复习词汇
  - 视频资源缺失
  - 数据加载中
- **风格**：扁平化 + 微立体质感，可参考 [Undraw](https://undraw.co/) 或 [Storyset](https://storyset.com/) 风格

**Emoji 辅助：**
- 在合适位置使用 Emoji 增强情感表达（如：🎉 学习完成、📚 开始阅读、⭐ 完美得分）

##### 2.2.3 字体系统
**中文字体：**
- **标题/强调**：思源黑体 (Noto Sans SC) - Bold
- **正文**：思源黑体 (Noto Sans SC) - Regular
- **数字/统计**：DIN Alternate / Inter（现代感更强）

**英文字体：**
- **标题**：Poppins / Outfit - SemiBold
- **正文/单词**：Inter / Roboto - Regular
- **音标**：专用音标字体如 Doulos SIL

**可读性优化：**
- 正文字号：16px → 17px（阅读更舒适）
- 行高：1.5 → 1.7（增加呼吸感）
- 字间距：0.5px（中英文混排优化）

##### 2.2.4 组件风格迭代

**卡片 (Card)：**
```
- 圆角：8px → 16px（更柔和）
- 阴影：从硬边阴影改为柔和扩散阴影
  box-shadow: 0 2px 8px rgba(0,0,0,0.08) 
  → 0 4px 20px rgba(0,0,0,0.06)
- 悬停效果：轻微上浮 + 阴影加深
```

**按钮 (Button)：**
```
- 主按钮：采用渐变背景 + 微光效果
- 圆角：从 4px → 12px
- 增加 Loading 状态的骨架屏动画
- Hover 状态：轻微缩放 (scale: 1.02) + 阴影
```

**进度条/徽章：**
```
- 学习进度条：从单色条改为渐变彩色条
- 状态徽章：增加图标 + 使用 Chip 组件的圆润风格
```

##### 2.2.5 微动效设计
为提升交互趣味性，增加以下动效（需保持克制，避免过度）：

- **页面过渡**：淡入 + 轻微上移 (Fade + Slide)
- **单词点击**：波纹效果 + 弹窗弹性进入 (Elastic)
- **完成反馈**：Confetti 撒花动画（仅在高分完成时触发）
- **加载状态**：骨架屏（Skeleton）代替 Spinner

#### 2.3 实施优先级
- **P0（必须）**：色彩系统优化、字体优化、卡片/按钮风格迭代
- **P1（重要）**：空状态插图、微动效（核心流程）
- **P2（优化）**：Emoji 点缀、高级动效（Confetti）

### 🌍 需求二：国际化 (i18n) 支持

#### 2.4 功能目标
实现完整的中英文双语界面切换，为以下场景提供支持：
- 母语为英语的学习者使用产品学习中文
- 中国学生通过英文界面强化英语思维
- 海外推广和国际化需求

#### 2.5 技术方案

##### 2.5.1 技术选型
- **核心库**：`react-i18next` + `i18next`
- **命名空间**：按功能模块划分（common, home, vocabulary, article, settings）
- **语言检测**：优先读取用户设置 → 浏览器语言 → 默认中文

##### 2.5.2 语言文件结构
```
/src/locales/
  ├── zh/                     # 简体中文
  │   ├── common.json         # 公共文案（导航、按钮等）
  │   ├── home.json           # 首页
  │   ├── vocabulary.json     # 单词本
  │   ├── article.json        # 文章阅读
  │   └── settings.json       # 设置页
  └── en/                     # 英语
      ├── common.json
      ├── home.json
      ├── vocabulary.json
      ├── article.json
      └── settings.json
```

##### 2.5.3 需要翻译的内容范围

**必须翻译（P0）：**
- 所有 UI 固定文案（导航栏、按钮、标签、提示）
- 表单提示和验证信息
- 空状态提示文案
- 设置项说明文字

**部分翻译（P1）：**
- 功能引导文案
- 帮助文档/FAQ

**无需翻译（保持原样）：**
- 单词数据本身（如单词拼写、释义）
- 用户生成内容（历史文章）
- API Key、技术配置项

##### 2.5.4 语言切换入口设计

**位置：**
- 主导航栏右上角，紧邻设置图标
- 使用地球图标 (🌐) + 当前语言文字

**交互方式：**
方案 A（推荐）：点击打开下拉菜单
```
🌐 中文 ▼
   → 简体中文 ✓
   → English
```

方案 B（简洁）：直接切换按钮
```
中文 / EN  （点击切换）
```

**持久化：**
- 用户选择的语言保存到 `localStorage`
- 刷新页面后自动应用用户上次选择

##### 2.5.5 特殊场景处理

**日期时间格式：**
- 中文：2025年11月29日 14:30
- 英文：Nov 29, 2025, 2:30 PM

**数字格式：**
- 统一使用阿拉伯数字（中英文通用）

**单位：**
- "天"/"days"、"个"/"words" 需根据语言动态调整

#### 2.6 实施检查清单

- [ ] 创建语言文件目录结构和初始 JSON 文件
- [ ] 在 `App.tsx` 初始化 i18n 实例
- [ ] 替换所有硬编码中文文案为 `t('key')` 调用
- [ ] 实现语言切换 UI 组件
- [ ] 测试所有页面的双语显示效果
- [ ] 补充遗漏的翻译条目

### 📖 需求三：文章阅读页排版优化

#### 2.7 当前问题诊断
基于 V1.3 的阅读页面，可能存在以下体验问题：

**排版问题：**
- 行宽过宽或过窄，影响阅读舒适度
- 段落间距不够，视觉拥挤
- 核心词高亮方式单一（仅加粗）

**交互问题：**
- 侧边栏/题目区与文章区布局不合理
- 缺少阅读辅助功能（如字号调节）
- 点击单词查看释义的视觉反馈不够明显

#### 2.8 优化方案

##### 2.8.1 核心阅读区排版优化

**容器宽度控制：**
```css
/* 最佳阅读宽度（基于认知科学研究）*/
.article-content {
  max-width: 680px;        /* 英文：50-75个字符/行 */
  margin: 0 auto;          /* 居中 */
  padding: 48px 32px;
}
```

**段落样式：**
```css
.article-content p {
  font-size: 17px;
  line-height: 1.8;        /* 行高加大 */
  margin-bottom: 20px;      /* 段间距 */
  text-align: justify;      /* 两端对齐（英文）*/
  text-justify: inter-word;
}
```

**核心词高亮优化：**
当前：仅使用 `<strong>` 加粗
优化：多层次视觉反馈
```tsx
// 方案 A：带底色高亮
<span className="target-word" 
      style={{
        fontWeight: 600,
        color: '#1976d2',
        background: 'linear-gradient(180deg, transparent 60%, #E3F2FD 60%)',
        cursor: 'pointer',
        padding: '2px 0'
      }}>
  {word}
</span>

// 方案 B：下划线 + 微标记
<span className="target-word">
  {word}
  <sup style={{color: '#FF6B6B', fontSize: '10px'}}>•</sup>
</span>
```

**首字母装饰（可选）：**
为文章首字母添加 Drop Cap 效果（像传统书籍）
```css
.article-content p:first-of-type::first-letter {
  float: left;
  font-size: 3.5em;
  line-height: 0.9;
  margin: 0.1em 0.1em 0 0;
  color: #4A90E2;
  font-weight: bold;
}
```

##### 2.8.2 布局结构优化

**桌面端布局（≥1024px）：**
采用三栏式布局，充分利用宽屏空间

```
┌─────────────────────────────────────────────┐
│  [导航栏]                           [设置]  │
├─────────────────────────────────────────────┤
│  [侧边] │   [文章阅读区]   │   [工具栏]   │
│  信息栏 │                  │   - 字号调节  │
│  - 难度 │   max-width:     │   - 进度显示  │
│  - 核心 │   680px          │   - 快捷操作  │
│  词汇   │   居中           │               │
│  - 进度 │                  │               │
│  160px  │                  │   200px       │
└─────────────────────────────────────────────┘
```

**平板/移动端布局（<1024px）：**
采用单栏式，顺序展示
```
┌─────────────────┐
│  [导航栏折叠]  │
├─────────────────┤
│  [阅读信息卡片]│
│  - 难度/词数    │
├─────────────────┤
│  [文章阅读区]  │
│  全宽展开       │
├─────────────────┤
│  [工具浮动按钮]│
│  右下角         │
└─────────────────┘
```

##### 2.8.3 阅读辅助功能

**字号控制：**
在工具栏提供 A- / A / A+ 三档字号切换
```tsx
const fontSizes = {
  small: '15px',
  medium: '17px',   // 默认
  large: '19px'
}
```

**阅读进度指示：**
在页面顶部添加细线型进度条，随滚动实时更新
```tsx
<LinearProgress 
  variant="determinate" 
  value={readProgress} 
  sx={{ height: 2, position: 'fixed', top: 64, width: '100%' }}
/>
```

**夜间模式（可选）：**
提供深色背景阅读模式（针对长时间阅读场景）
- 背景：#1E1E1E
- 文字：#E0E0E0
- 核心词：#64B5F6

##### 2.8.4 单词交互优化

**点击反馈：**
```css
.target-word:hover {
  background: linear-gradient(180deg, transparent 55%, #BBDEFB 55%);
  transform: translateY(-1px);
  transition: all 0.2s ease;
}

.target-word:active {
  transform: translateY(0);
}
```

**弹窗优化：**
- 从屏幕中央弹窗改为**跟随点击位置**的 Popover（靠近点击词汇）
- 弹窗内容增加"收藏""标记已掌握"快捷操作
- 支持键盘 ESC 关闭

##### 2.8.5 引导元素

**首次阅读引导：**
第一次进入阅读页时，显示轻量级提示：
```
💡 提示：点击蓝色单词可查看释义和视频讲解
```
（3秒后自动淡出，或用户点击关闭后不再显示）

#### 2.9 实施优先级
- **P0（必须）**：核心阅读区排版优化、核心词高亮优化
- **P1（重要）**：布局结构优化、字号控制、阅读进度指示
- **P2（优化）**：夜间模式、Drop Cap 首字母、高级动效

## 3. 非功能性需求

### 3.1 性能要求
- 语言切换响应时间 < 200ms
- 阅读页首屏渲染时间 < 1s
- 动效帧率保持 60fps

### 3.2 兼容性要求
- 浏览器：Chrome 90+, Safari 14+, Edge 90+
- 屏幕尺寸：支持 768px - 2560px 宽度
- iPad 触控操作优化

### 3.3 可访问性 (A11y)
- 所有按钮提供 aria-label
- 支持键盘导航（Tab / Enter / ESC）
- 色彩对比度符合 WCAG 2.1 AA 标准

## 5. 风险与预案

### 5.1 潜在风险

**设计风险：**
- 风格调整可能引起老用户不适应
- **预案**：提供"经典模式"切换选项（保留 V1.3 风格）

**技术风险：**
- i18n 集成可能影响现有状态管理
- **预案**：逐步迁移，每个模块单独测试

**进度风险：**
- 设计细节打磨可能超期
- **预案**：采用 MoSCoW 优先级管理，确保 P0 功能按时完成

### 5.2 回滚策略
- 保留 V1.3 版本代码分支
- 使用 Feature Flag 控制新功能开关
- 提供版本降级按钮（在设置页）
