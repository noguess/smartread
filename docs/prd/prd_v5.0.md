## 1. 版本概述 (Overview)

V5.0 版本不涉及核心算法的重构，而是专注于**用户体验 (UX)** 的打磨。主要解决用户在等待文章生成时的“黑盒”焦虑，同时优化首页 UI 的视觉平衡，并根据实际反馈调整文章长度设置。

---

## 2. 详细需求规格 (Detailed Requirements)

### 🚀 需求一：文章生成流式展示 (Streaming Generation Feedback)

#### 2.1 现状问题
目前文章生成过程（尤其是 V4.0 加入复杂题型后）耗时较长。用户在 Loading 页面只能看到静态的旋转图标，无法感知后端是否在正常工作，容易产生焦虑或误以为卡死。

#### 2.2 解决方案
将 Loading 页面改造为**流式内容展示页**。

*   **实时反馈**:
    *   **理想方案**: 对接 LLM 的 Stream API，实时将生成的文章内容逐字/逐行打印在屏幕上。
    *   **兜底方案** (若 API 不支持): 采用分阶段进度提示 (Progress Steps)，如：“正在分析词汇... -> 正在构思大纲... -> 正在撰写正文... -> 正在生成题目...”。
*   **UI 设计**:
    *   背景可保持模糊或半透明。
    *   中央区域显示类似“打字机”效果的文本流。
    *   底部保留进度条或动态 Loading 动画。
*   **体验目标**: 让用户感觉到“AI 正在为你实时创作”，增加沉浸感和掌控感。

---

### 🎨 需求二：首页“开始学习”模块重构 (Homepage Redesign)

#### 2.3 视觉一致性优化
*   **问题**: 左侧“开始学习”模块与右侧“记忆状态”模块在高度、边距、阴影风格上不一致，导致页面割裂。
*   **要求**:
    *   统一左右两侧卡片的**高度 (Height)** 和 **圆角 (Border Radius)**。
    *   统一内部元素的**间距 (Padding/Margin)**。
    *   确保视觉重心平衡。

#### 2.4 学习统计替换 (Learning Stats)
*   **移除**: 原“今日计划”文本区域。
*   **新增**: **学习情况统计**。
    *   **场景 A: 连续学习中**
        *   显示文案：“已连续学习 **N** 天”。
        *   显示数据：“累计学习时长 **M** 分钟”。
        *   视觉建议：使用醒目的数字字体，搭配“火焰”或“时钟”图标。
    *   **场景 B: 断签 (未连续)**
        *   显示文案：“上次学习时间：**YYYY-MM-DD**”。
        *   显示鼓励语：“加油，持续学习才能进步！”或“别灰心，今天重新开始！”。

#### 2.5 “自定义模式”入口优化
*   **问题**: 原“自定义模式”仅为文字链接，点击感弱，易被忽略。
*   **要求**:
    *   将“自定义模式”升级为**次级按钮 (Secondary Button)** 或 **独立的小卡片**。
    *   与主按钮“智能生成 (Smart Generate)”形成层级区分，但必须具备明确的**按钮形态**（有边框、背景色或 Hover 效果）。
    *   布局上可置于“智能生成”按钮下方或旁侧，确保易于点击。

---

### ⚙️ 需求三：设置页偏好调整 (Settings Update)

#### 2.6 文章长度参数调整
用户反馈原有字数偏少，V5.0 将大幅提升文章长度标准，以适应更高强度的阅读训练。

*   **Short (短篇)**:
    *   原: ~150 词
    *   **新: 400 词**
*   **Medium (中篇 - 默认)**:
    *   原: ~250 词
    *   **新: 600 词**
*   **Long (长篇)**:
    *   原: ~350+ 词
    *   **新: 800 词**

> **注意**: 需同步更新 LLM 的 System Prompt，明确字数要求（如 `Length: approximately 600 words`），并确保生成的文章逻辑通顺，避免单纯堆砌字数。

## 3. 交互与 UI 细节 (UI/UX Details)

### 3.1 Loading 页面
*   **动画**: 建议使用 Lottie 动画或 CSS 打字机效果。
*   **文案**: 动态变化，如 "AI is thinking...", "Drafting the story...", "Creating quiz..."。

### 3.2 首页布局
*   建议采用 Grid 布局，确保左右两栏在不同屏幕尺寸下的响应式表现。
*   “学习统计”区域可使用带有微渐变的背景，突出数据的成就感。

---

## 4. 实施计划 (Implementation Plan)

1.  **P0 - 设置更新**: 修改 `Settings` 定义及相关 Prompt 逻辑（字数调整）。
2.  **P1 - 首页重构**: 修改 `Home` 组件，调整 CSS 样式，实现统计逻辑和新按钮样式。
3.  **P1 - Loading 优化**: 改造 `GeneratingView`，尝试接入 Stream 流或优化进度展示。
4.  **P2 - 统计逻辑**: 在 `History` 或 `User` 数据中计算“连续学习天数”和“累计时长”。
